Paystack setup

Connect live payments without exposing secret keys.

This guide is for LearnedCircle admin use. It keeps the payment launch sequence clear before premium, advert and client-lawyer work payments are made available to users.

Environment variables

Add the Paystack key in Vercel.

Add the secret key only in Vercel Project Settings. Do not paste Paystack secret keys into public HTML, JavaScript, GitHub issues or chat messages.

Production: PAYSTACK_SECRET_KEY Testing: PAYSTACK_TEST_SECRET_KEY can be used before the live key is ready. After adding a key: redeploy the Vercel project so the API route can read it.

Paystack dashboard URLs

Use these settings when configuring Paystack.

Webhook URL: https://learnedcircle.com/api/account-auth Callback route: LearnedCircle sends users back to /account.html?payment=paystack&reference=... Account test page: https://learnedcircle.com/account.html?mode=login

Test checklist

Confirm these before accepting live payments.

Log in as a test lawyer and start a monthly premium checkout. Complete one successful Paystack test payment and confirm the account shows premium pending. Check Supabase platform_payments for a paid record with the Paystack reference. Check Supabase premium_subscriptions for the linked premium subscription. Run one failed or abandoned payment and confirm the account does not become premium. After live key activation, repeat the same test with a small real transaction before public launch.
